# 模拟面试模块功能概要

## 当前状态

### 已有后端能力
- `POST /interview/analyse`
  - 根据简历 PDF 地址提取文本
  - 用本地标签池做关键词匹配
- `POST /interview/start`
  - 根据选中的关键词开始出题
- `POST /interview/submit`
  - 提交作答并生成评分结果
- `POST /interview/getHistory`
  - 分页查询面试记录
- `GET /interview/detail`
  - 查询某次面试的题目详情

### 当前引擎
- `JI_CHI`
  - 本地规则模式
  - 关键词来自 `subject_label`
  - 题目来自 `subject_info + subject_mapping + subject_brief`
  - 评分和评语来自本地工具类
- `ALI_BL`
  - 大模型模式
  - 支持模型出题和模型评分
  - 现已改成从配置读取 `api-key / endpoint / model / prompt`

### 当前前端现状
- `front-ape-club` 已补齐 `/inter-list` 模拟面试页面
- 顶部菜单“模拟面试”已可直接进入
- 当前前端已支持简历上传、关键词选择、开始面试、作答、提交和历史记录查看

## 数据库设计

### `sql/interview.sql` 原问题
- 两条建表语句之间缺少分号
- 最后一条建表语句末尾也缺少分号
- 这是明确的 SQL 语法错误

### 这次补充后的字段

#### `interview_history`
- `engine_type`
  - 记录本次面试使用的引擎，如 `JI_CHI`、`ALI_BL`
- `avg_score`
  - 本次平均分
- `avg_tip`
  - 本次平均分结论，例如“掌握较差 / 掌握程度 ok / 掌握程度很好”
- `key_words`
  - 本次命中的标签关键词
- `tip`
  - 本次面试的总体评价
- `interview_url`
  - 简历地址
- `question_count`
  - 本次抽题数量

#### `interview_question_history`
- `interview_id`
  - 所属面试场次
- `question_order`
  - 第几题
- `engine_type`
  - 当前题目来源引擎
- `label_name`
  - 当前题目的标签
- `score`
  - 当前题最终得分
- `question / answer / user_answer`
  - 改成 `text`，避免内容被截断

### 为什么要补这些字段
- 后面如果本地题库和大模型混合出题，必须能追溯“题从哪里来的”
- 历史记录页不能只看平均分，还要知道题量和总评
- 详情页需要顺序和标签，不然展示层只能靠字符串拼接

## 建议的产品流程

### 1. 上传简历
1. 前端上传 PDF 到 OSS
2. OSS 返回可访问地址
3. 页面左侧展示 PDF 预览

### 2. 简历分析
1. 后端下载 PDF 并提取文本
2. 先用本地标签池做快速匹配
3. 如果启用大模型，再用模型补充和去重关键词
4. 将推荐标签返回前端，用户可手动勾选和修正

### 3. 开始面试
1. 先从本地题库抽题
2. 如果本地题库题量不足，再由大模型补题
3. 控制题量在 `6~8` 题
4. 每题返回：
   - 标签名
   - 题目内容
   - 标准答案
   - 关键词

### 4. 作答过程
1. 用户先输入自己的回答
2. 未输入回答时不允许提交分析
3. 支持查看参考答案和解析
4. 评分支持两种模式：
   - 本地模式：用户自评
   - AI 模式：模型评分 + 参考答案

### 5. 提交分析
1. 保存每道题的答题明细
2. 汇总平均分
3. 生成总体评价
4. 生成分标签评价
5. 保存历史记录供分页查询和详情回放

## 大模型接入方案

### 当前问题
- 原代码把大模型 key 写死在 `AlBLInterviewEngine` 中
- prompt 太短，输出格式不稳定
- 解析逻辑过度依赖 ```json 代码块

### 本次优化
- 改为从配置读取：
  - `endpoint`
  - `model`
  - `api-key`
  - `question-count`
  - `prompt`
- 配置支持环境变量注入
- 大模型调用改成兼容 OpenAI 的 `Chat Completions API`
- prompt 改成：
  - `system prompt`
  - `user prompt template`
- 统一要求模型只返回 JSON
- 增加 JSON 解析兜底

### 配置示例
```yaml
interview:
  ai:
    endpoint: https://dashscope.aliyuncs.com/compatible-mode/v1/chat/completions
    model: qwen3.5-flash
    api-key: ${DASHSCOPE_API_KEY:}
    question-count: 8
    prompt:
      question-system: >
        你是一名资深技术面试官。请围绕给定关键词生成一条真实、简洁、适合口头追问的面试题。
        只能返回 JSON，不要返回 markdown、解释说明或多余文本。
      question-user-template: >
        请围绕关键词“%s”生成 1 道面试题。
        返回 JSON：{"labelName":"标签名称","subjectName":"题目内容"}
      score-system: >
        你是一名严谨的技术面试官。请根据题目和候选人回答进行评分，并给出参考答案。
        只能返回 JSON，不要返回 markdown、解释说明或多余文本。
      score-user-template: >
        请根据下面信息完成评分，满分 5 分，可保留 1 位小数。
        题目：%s
        候选人回答：%s
        返回 JSON：{"userScore":4.5,"subjectAnswer":"参考答案简述"}
```

## 推荐 Prompt

### 出题 Prompt
#### system
你是一名资深技术面试官。请围绕给定关键词生成一条真实、简洁、适合口头追问的面试题。只能返回 JSON，不要返回 markdown、解释说明或多余文本。

#### user
请围绕关键词“%s”生成 1 道面试题。  
返回 JSON：`{"labelName":"标签名称","subjectName":"题目内容"}`

### 评分 Prompt
#### system
你是一名严谨的技术面试官。请根据题目和候选人回答进行评分，并给出参考答案。只能返回 JSON，不要返回 markdown、解释说明或多余文本。

#### user
请根据下面信息完成评分，满分 5 分，可保留 1 位小数。  
题目：`%s`  
候选人回答：`%s`  
返回 JSON：`{"userScore":4.5,"subjectAnswer":"参考答案简述"}`
